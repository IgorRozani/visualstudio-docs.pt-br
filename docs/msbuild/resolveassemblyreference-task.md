# Tarefa ResolveAssemblyReference
[!INCLUDE[vs2017banner](../code-quality/includes/vs2017banner.md)]

Determina todos os assemblies que dependem dos assemblies especificados. Isso inclui o segundo e `n`dependências\-ordem.  
  
## Parâmetros  
 A tabela a seguir descreve os parâmetros do `ResolveAssemblyReference` tarefa.  
  
|Parâmetro|Descrição|  
|---------------|---------------|  
|`AllowedAssemblyExtensions`|Opcional `String[]` parâmetro.<br /><br /> As extensões de nome de arquivo de assembly para usar ao resolver referências. As extensões de nome de arquivo padrão são .exe e. dll.|  
|`AllowedRelatedFileExtensions`|Opcional `String[]` parâmetro.<br /><br /> As extensões de nome de arquivo a ser usado para uma pesquisa de arquivos que estão relacionados uns aos outros. As extensões padrão são. PDB e. XML.|  
|`AppConfigFile`|Opcional `String` parâmetro.<br /><br /> Especifica um arquivo App. config analisar e extrair bindingRedirect mapeamentos. Se esse parâmetro for especificado, o `AutoUnify` parâmetro deve ser `false`.|  
|`AutoUnify`|Opcional `Boolean` parâmetro.<br /><br /> Esse parâmetro é usado para criar assemblies, como DLLs, que não podem ter um arquivo App. config normal.<br /><br /> Quando `true`, o gráfico de dependência resultante é tratado como se houvesse anApp.Config arquivo passado para o parâmetro AppConfigFile automaticamente. Esse arquivo App. config virtual tem uma entrada de bindingRedirect para cada conjunto conflitante de assemblies, de modo que o assembly de versão mais alto é escolhido. Uma conseqüência disso é que nunca haverá um aviso sobre assemblies conflitantes porque cada conflito será foram resolvido.<br /><br /> Quando `true`, cada remapeamento distintos resultará em um comentário de alta prioridade, mostrando as versões nova e antiga e que `AutoUnify` foi `true`.<br /><br /> Quando `true`, o parâmetro AppConfigFile deve estar vazio<br /><br /> Quando `false`, nenhuma versão do assembly remapeamento ocorrerá automaticamente. Quando existem duas versões de um assembly, um aviso será emitido.<br /><br /> Quando `false`, cada conflito distinto entre versões diferentes dos mesmos resultados de assembly em um comentário de alta prioridade. Esses comentários são seguidos por um único aviso. O aviso tem um código de erro exclusivo e contém o texto "Encontrados conflitos entre diferentes versões de referência e assemblies dependentes".|  
|`Assemblies`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro.<br /><br /> Especifica os itens para os quais as dependências e caminhos completos devem ser identificadas. Esses itens podem ter nomes simples como "Sistema" ou nomes fortes, como "System, Version \= 2.0.3500.0, Culture \= neutral, PublicKeyToken \= b77a5c561934e089."<br /><br /> Itens passados para esse parâmetro podem opcionalmente ter os metadados de item a seguir:<br /><br /> -   `Private`: `Boolean` valor. Se `true`, em seguida, o item será copiado localmente. O valor padrão é `true`.<br />-   `HintPath`: `String` valor. Especifica o caminho e nome de arquivo para usar como referência. Isso é usado quando {HintPathFromItem} é especificado no `SearchPaths` parâmetro. O valor padrão é uma cadeia de caracteres vazia.<br />-   `SpecificVersion`: `Boolean` valor. Se `true`, em seguida, o nome exato especificado no `Include` atributo deve corresponder. Se `false`, qualquer assembly com o mesmo nome simple funcionará. Se `SpecificVersion` não for especificado, então a tarefa examina o valor de `Include` atributo do item. Se o atributo é um nome simples, ele se comporta como se `SpecificVersion` foi `false`. Se o atributo for um nome forte, ele se comporta como se `SpecificVersion` foi `true`.<br />     Quando usado com um tipo de item de referência, o `Include` atributo deve ser o nome completo de fusão do assembly a ser resolvido. O assembly é resolvido apenas se corresponder exatamente a fusão de `Include` atributo.<br />     Quando um projeto tem como alvo uma versão do .NET Framework e referencia um assembly compilado para uma versão posterior do .NET Framework, a referência resolve apenas se ele tem `SpecificVersion` definido como `true`.<br />     Quando um projeto tem como alvo um perfil e referencia um assembly que não esteja no perfil, a referência resolve apenas se ele tem `SpecificVersion` definido como `true`.<br />-   `ExecutableExtension`: `String` valor. Quando presente, o assembly resolvido deve ter essa extensão. Quando ausente,. dll é considerado primeiro, seguido por .exe, para cada diretório examinado.<br />-   `SubType`: `String` valor. Somente os itens com metadados de subtipo vazio serão resolvidos em caminhos do assembly completo. Itens com metadados de subtipo não vazias são ignorados.<br />-   `AssemblyFolderKey`: `String` valor. Esses metadados é suportado para fins de legado. Especifica uma chave de registro definidas pelo usuário, como "hklm\\*VendorFolder*", que `Assemblies` deve usar para resolver referências de assembly.|  
|`AssemblyFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro.<br /><br /> Especifica uma lista de assemblies totalmente qualificados para encontrar as dependências.<br /><br /> Itens passados para esse parâmetro podem opcionalmente ter os metadados de item a seguir:<br /><br /> -   `Private`: opcional `Boolean` valor. Se for true, o item será copiado localmente.<br />-   `FusionName`: opcional `String` metadados. Especifica o nome forte ou simple para este item. Se esse atributo estiver presente, isso pode economizar tempo porque o arquivo de assembly não precisa ser aberto para obter o nome.|  
|`AutoUnify`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, o gráfico de dependência resultante é tratado como se houvesse um arquivo App. config passado para o parâmetro AppConfigFile automaticamente. Esse arquivo App. config virtual tem uma entrada de bindingRedirect para cada conjunto conflitante de assemblies para que o assembly de versão mais alto é escolhido. Um resultado disso é que nunca haverá um aviso sobre assemblies conflitantes porque cada conflito será foram resolvido. Cada remapeamento distintos causará um comentário de alta prioridade que indica as versões antigas e novas e o fato de que isso era feito automaticamente porque `AutoUnify` foi `true`.<br /><br /> Se `false`, nenhuma versão do assembly remapeamento ocorrerá automaticamente. Quando existem duas versões de um assembly, haverá um aviso. Cada conflito distinto entre versões diferentes do mesmo assembly fará com que um comentário de alta prioridade. Depois que todos esses comentários são exibidos, haverá um único aviso com um único erro código e texto "encontrados conflitos entre diferentes versões de referência e assemblies dependentes".<br /><br /> O valor padrão é `false`.|  
|`CandidateAssemblyFiles`|Opcional `String[]` parâmetro.<br /><br /> Especifica uma lista de módulos \(assemblies\) a ser usado para o processo de pesquisa e resolução. Valores passados para este parâmetro devem ser nomes de arquivo absoluto ou relativo ao projeto de arquivo.<br /><br /> Assemblies nesta lista serão considerados quando o `SearchPaths` parâmetro contém {CandidateAssemblyFiles} como um dos caminhos a serem considerados.|  
|`CopyLocalDependenciesWhenParentReferenceInGac`|Opcional [Boolean](assetId:///Boolean?qualifyHint=False&autoUpgrade=True) parâmetro.<br /><br /> Se for verdadeiro determinar se uma dependência deve ser copiada localmente, uma das verificações feitas ver se a referência do pai no arquivo de projeto tem metadados particular definido. Se o conjunto, em seguida, o valor particular é usado como uma dependência.<br /><br /> Se os metadados não estiver definido, então a dependência passa pelas mesmas verificações de referência pai. Uma dessas verificações é verificar se a referência está no GAC. Se for uma referência no GAC, em seguida, ele não é copiado localmente, porque ele é considerado no GAC no computador de destino. Aplica\-se somente a uma referência específica e não suas dependências.<br /><br /> Por exemplo, uma referência no arquivo de projeto que está no GAC não é copiada localmente, mas suas dependências são copiadas localmente porque eles não estão no GAC.<br /><br /> Se falso, as referências de arquivo de projeto são verificadas para ver se elas estão no GAC e são copiadas localmente, conforme apropriadas.<br /><br /> As dependências são verificadas para ver se eles estão no GAC e também são verificadas para ver se a referência do pai do arquivo de projeto está no GAC.<br /><br /> Se a referência do pai do arquivo de projeto está no GAC, a dependência não é copiada localmente.<br /><br /> Se esse parâmetro é verdadeiro ou falso, se houver várias referências do pai e qualquer uma delas não estão no GAC, todas elas são copiadas localmente.|  
|`CopyLocalFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Retorna todos os arquivos de `ResolvedFiles`, `ResolvedDependencyFiles`, `RelatedFiles`, `SatelliteFiles`, e `ScatterFiles` parâmetros tem `CopyLocal` metadados de item com um valor de `true`.|  
|`FilesWritten`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída.<br /><br /> Contém os itens gravados em disco.|  
|`FindDependencies`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, encontrará dependências. Caso contrário, apenas referências principais são encontradas. O valor padrão é `true`.|  
|`FindRelatedFiles`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, relacionadas a arquivos, como arquivos. PDB e arquivos. XML serão encontrados. O valor padrão é `true`.|  
|`FindSatellites`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, assemblies satélite serão encontrados. O valor padrão é `true.`|  
|`FindSerializationAssemblies`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, em seguida, a tarefa procura assemblies de serialização. O valor padrão é `true`.|  
|`FullFrameworkAssemblyTables`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro.<br /><br /> Especifica os itens que têm metadados de "FrameworkDirectory" para associar uma lista redist diretório uma estrutura específica. Se a associação não é feita, um erro será registrado. A lógica de referência \(RAR\) resolver assembly usa o diretório do framework de destino se um FrameworkDirectory não está definida...|  
|`FullFrameworkFolders`|Opcional [String](assetId:///String?qualifyHint=False&autoUpgrade=True)`[]` parâmetro.<br /><br /> Especifica o conjunto de pastas que contêm um diretório RedistList. Esse diretório representa a estrutura completa para um perfil de cliente, por exemplo, %programfiles%\\reference assemblies\\microsoft\\framework\\v4.0.|  
|`FullTargetFrameworkSubsetNames`|Opcional `String[]` parâmetro.<br /><br /> Contém uma lista de nomes de subconjunto de estrutura de destino. Se um nome de subconjunto na lista corresponde a um no `TargetFrameworkSubset` nome de propriedade, em seguida, o sistema exclui esse subconjunto de estrutura de destino específico no tempo de compilação.|  
|`IgnoreDefaultInstalledAssemblyTables`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, em seguida, a tarefa de pesquisa e usos adicionais instalados assembly tabelas \(ou "Lista Redist"\) que está localizado no diretório \\RedistList em `TargetFrameworkDirectories`. O valor padrão é `false.`|  
|`IgnoreDefaultInstalledAssemblySubsetTables`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, em seguida, a tarefa de pesquisa e usos adicionais instalados assembly tabelas de subconjunto \(ou "Lista de subconjunto"\) que está localizado no diretório \\SubsetList em `TargetFrameworkDirectories`. O valor padrão é `false.`|  
|`InstalledAssemblySubsetTables`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro.<br /><br /> Contém uma lista de arquivos XML que especificam os assemblies que devem estar no subconjunto do destino.<br /><br /> Como opção, os itens nessa lista podem especificar os metadados de "FrameworkDirectory" para associar um `InstalledAssemblySubsetTable`<br /><br /> com um diretório de estrutura específico.<br /><br /> Se houver apenas um `TargetFrameworkDirectories` elemento e, em seguida, todos os itens da lista que não possuem os metadados de "FrameworkDirectory" são tratados como se eles são definidos como o valor exclusivo que é passado para `TargetFrameworkDirectories`.|  
|`InstalledAssemblyTables`|Opcional `String` parâmetro.<br /><br /> Contém uma lista de arquivos XML que especificam os assemblies que devem ser instalados no computador de destino.<br /><br /> Quando `InstalledAssemblyTables` estiver definido, as versões anteriores dos assemblies na lista são mescladas em versões mais recentes listados no XML. Além disso, os assemblies que têm uma configuração de InGAC \= 'true' são considerados pré\-requisitos e definir a CopyLocal \= 'false', a menos que explicitamente substituída.<br /><br /> Como opção, os itens nessa lista podem especificar metadados de "FrameworkDirectory" para associar um `InstalledAssemblyTable` com um diretório de estrutura específico.  No entanto, essa configuração será ignorada, a menos que o nome do pacote redistribuível começa com<br /><br /> "Microsoft\-Windows\-CLRCoreComp".<br /><br /> Se houver apenas um `TargetFrameworkDirectories` elemento e, em seguida, todos os itens da lista que não possuem os metadados de "FrameworkDirectory" são tratados como se eles são definidos como o valor exclusivo que é passado<br /><br /> para `TargetFrameworkDirectories`.|  
|`LatestTargetFrameworkDirectories`|Opcional `String[]` parâmetro.<br /><br /> Especifica uma lista de diretórios que contêm listas redist para o framework mais recente que podem ser direcionados para a máquina. Se isso não for definido o mais alto framework instalado no computador para um identificador de estrutura de destino fornecido é usado.|  
|`ProfileName`|Opcional assetId:///String?qualifyHint=False&autoUpgrade=True parâmetro.<br /><br /> -   Especifica o nome do perfil do framework sejam direcionadas. Por exemplo, cliente, Web ou rede.|  
|`RelatedFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém os arquivos relacionados, como arquivos XML e. PDB que têm o mesmo nome de base como uma referência.<br /><br /> Os arquivos listados nesse parâmetro opcionalmente podem conter os metadados de item a seguir:<br /><br /> -   `Primary`: `Boolean` valor. Se `true`, em seguida, o item do arquivo passado para a matriz, usando o `Assemblies` parâmetro. O valor padrão é `false`.<br />-   `CopyLocal`: `Boolean` valor. Indica se a referência fornecida deve ser copiada para o diretório de saída.|  
|`ResolvedDependencyFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém o *n*caminhos de ordem th dependências. Esse parâmetro não inclui o primeiro referências principais ordem, que estão contidas no `ResolvedFiles` parâmetro.<br /><br /> Os itens nesse parâmetro, opcionalmente, contenham os metadados de item a seguir:<br /><br /> -   `CopyLocal`: `Boolean` valor. Indica se a referência fornecida deve ser copiada para o diretório de saída.<br />-   `FusionName`: `String` valor. Especifica o nome para essa dependência.<br />-   `ResolvedFrom`: `String` valor. Especifica o caminho de pesquisa literal que este arquivo foi resolvido.|  
|`ResolvedFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém uma lista de todas as referências primárias resolvido para caminhos completos.<br /><br /> Os itens nesse parâmetro, opcionalmente, contenham os metadados de item a seguir:<br /><br /> -   `CopyLocal`: `Boolean` valor. Indica se a referência fornecida deve ser copiada para o diretório de saída.<br />-   `FusionName`: `String` valor. Especifica o nome para essa dependência.<br />-   `ResolvedFrom`: `String` valor. Especifica o caminho de pesquisa literal que este arquivo foi resolvido.|  
|`SatelliteFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Especifica todos os arquivos satélite encontrados. Esses serão CopyLocal \= true se a referência ou dependência que causou este item para existir é CopyLocal \= true.<br /><br /> Os itens nesse parâmetro, opcionalmente, contenham os metadados de item a seguir:<br /><br /> -   `CopyLocal`: `Boolean` valor. Indica se a referência fornecida deve ser copiada para o diretório de saída. Esse valor é `true` se a referência ou dependência que causou este item para existir tem um `CopyLocal` valor `true`.<br />-   `DestinationSubDirectory`: `String` valor. Especifica o diretório de destino relativo para copiar este item.|  
|`ScatterFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém os arquivos de dispersão associados a um dos assemblies.<br /><br /> Os itens nesse parâmetro, opcionalmente, contenham os metadados de item a seguir:<br /><br /> -   `CopyLocal`: `Boolean` valor. Indica se a referência fornecida deve ser copiada para o diretório de saída.|  
|`SearchPaths`|Necessário `String[]` parâmetro.<br /><br /> Especifica as pastas ou locais especiais que são pesquisados para localizar os arquivos no disco que representam os assemblies. A ordem na qual os caminhos de pesquisa são listados é importante. Para cada assembly, a lista de caminhos é pesquisada da esquerda para a direita. Quando um arquivo que representa o assembly for encontrado, que pesquisa será interrompido e procurar a próxima começa do assembly.<br /><br /> Esse parâmetro aceita os seguintes tipos de valores:<br /><br /> -   Um caminho de diretório.<br />-   {HintPathFromItem}: Especifica que a tarefa examinará o `HintPath` metadados do item base.<br />-   {CandidateAssemblyFiles}: Especifica que a tarefa examinará os arquivos transmitidos por meio de `CandidateAssemblyFiles` parâmetro.<br />-   {Do registro: \_AssemblyFoldersBase\_, \_RuntimeVersion\_, \_AssemblyFoldersSuffix\_}:<br />-   {AssemblyFolders}: Especifica a tarefa usará o esquema de localizar assemblies de registro do Visual Studio.NET 2003.<br />-   {GAC}: Especifica a tarefa irá procurar no GAC.<br />-   {RawFileName}: Especifica a tarefa considerará o `Include` valor do item a ser um nome de arquivo e caminho exato.|  
|`SerializationAssemblyFiles`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém os assemblies de serialização XML encontrados. Esses itens são marcados CopyLocal \= true se e somente se a referência ou dependência que causou este item para existir é CopyLocal \= true.<br /><br /> O `Boolean` metadados CopyLocal indica se a referência fornecida deve ser copiada para o diretório de saída.|  
|`Silent`|Opcional `Boolean` parâmetro.<br /><br /> Se `true`, não as mensagens são registradas. O valor padrão é `false`.|  
|`StateFile`|Opcional `String` parâmetro.<br /><br /> Especifica um nome de arquivo que indica onde salvar intermediários criar estado para esta tarefa.|  
|`SuggestedRedirects`|Opcional <xref:Microsoft.Build.Framework.ITaskItem>`[]` parâmetro de saída somente leitura.<br /><br /> Contém um item para cada distintos identidade de assembly em conflito, independentemente do valor de `AutoUnify` parâmetro. Isso inclui cada cultura e PKT encontrado que não tinha uma entrada bindingRedirect adequado no arquivo de configuração do aplicativo.<br /><br /> Opcionalmente, cada item contém as seguintes informações:<br /><br /> -   `Include` atributo: contém o nome completo da família do assembly com um valor de campo de versão de 0.0.0.0<br />-   `MaxVersion` metadados de item: contém o número máximo de versão.|  
|`TargetedRuntimeVersion`|Opcional `String` parâmetro.<br /><br /> Especifica a versão de tempo de execução para o destino, por exemplo, 2.0.57027 ou v2.0.57027.|  
|`TargetFrameworkDirectories`|Opcional `String[]` parâmetro.<br /><br /> Especifica o caminho do diretório do framework de destino. Este parâmetro é necessário para determinar o status de CopyLocal para itens resultantes.<br /><br /> Se esse parâmetro não for especificado, nenhum item resultante será ter valor CopyLocal `true` a menos que explicitamente tenham um `Private` valor de metadados de `true` em seu item de origem.|  
|`TargetFrameworkMoniker`|Opcional `String` parâmetro.<br /><br /> TargetFrameworkMoniker para monitorar, se houver. Isso é usado para fazer logon.|  
|`TargetFrameworkMonikerDisplayName`|Opcional `String` parâmetro.<br /><br /> O nome de exibição de TargetFrameworkMoniker para monitorar, se houver. Isso é usado para fazer logon.|  
|`TargetFrameworkSubsets`|Opcional `String[]` parâmetro.<br /><br /> Contém uma lista de nomes de subconjunto de estrutura de destino a ser procurado em diretórios do framework de destino.|  
|`TargetFrameworkVersion`|Opcional `String` parâmetro.<br /><br /> Versão do framework de destino do projeto. O valor padrão é vazio, o que significa que não há nenhuma filtragem para as referências com base na estrutura de destino.|  
|`TargetProcessorArchitecture`|Opcional `String` parâmetro.<br /><br /> A arquitetura do processador de destino preferencial. Usado para resolver referências de Cache de Assembly Global \(GAC\).<br /><br /> Esse parâmetro pode ter um valor de `x86`, `IA64` ou `AMD64`.<br /><br /> Se esse parâmetro estiver ausente, a tarefa considera primeiro assemblies que correspondem à arquitetura do processo em execução no momento. Se nenhum assembly for encontrada, a tarefa considera assemblies no GAC que têm `ProcessorArchitecture` valor `MSIL` ou nenhuma `ProcessorArchitecture` valor.|  
  
## Avisos  
 Os seguintes avisos são registrados:  
  
-   `ResolveAssemblyReference.TurnOnAutoGenerateBindingRedirects`  
  
-   `ResolveAssemblyReference.SuggestedRedirects`  
  
-   `ResolveAssemblyReference.FoundConflicts`  
  
-   `ResolveAssemblyReference.AssemblyFoldersExSearchLocations`  
  
-   `ResolveAssemblyReference.UnifiedPrimaryReference`  
  
-   `ResolveAssemblyReference.PrimaryReference`  
  
-   `ResolveAssemblyReference.UnifiedDependency`  
  
-   `ResolveAssemblyReference.UnificationByAutoUnify`  
  
-   `ResolveAssemblyReference.UnificationByAppConfig`  
  
-   `ResolveAssemblyReference.UnificationByFrameworkRetarget`  
  
## Comentários  
 Além dos parâmetros listados acima, essa tarefa herda parâmetros do <xref:Microsoft.Build.Tasks.TaskExtension> classe, que herda de <xref:Microsoft.Build.Utilities.Task> classe. Para obter uma lista desses parâmetros adicionais e suas descrições, consulte [Classe TaskExtension \(base\)](../msbuild/taskextension-base-class.md).
