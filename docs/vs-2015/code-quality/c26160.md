# <a name="c26160"></a>C26160
[!INCLUDE[vs2017banner](../includes/vs2017banner.md)]

A versão mais recente deste tópico pode ser encontrada em [C26160](https://docs.microsoft.com/visualstudio/code-quality/c26160).  
  
Aviso C26160: possível falha ao manter bloqueio do chamador \<lock > antes de chamar a função \<func >.  
  
 Aviso C26160 se parece com aviso [C26110](../code-quality/c26110.md) , exceto que o nível de confiança é menor. Por exemplo, a função pode conter erros de anotação.  
  
## <a name="example"></a>Exemplo  
 O código a seguir gera o aviso C26160.  
  
```cpp  
  
struct Account  
{  
    _Guarded_by_(cs) int balance;  
    CRITICAL_SECTION cs;  
  
    _No_competing_thread_ void Init()   
    {  
        balance = 0; // OK  
    }  
  
    _Requires_lock_held_(this->cs) void FuncNeedsLock();  
  
    _No_competing_thread_ void FuncInitCallOk()  
        // this annotation requires this function is called   
        // single-threaded, therefore we don't need to worry   
        // about the lock  
    {  
        FuncNeedsLock(); // OK, single threaded  
    }   
  
    void FuncInitCallBad() // No annotation provided, analyzer generates warning  
    {  
        FuncNeedsLock(); // Warning C26160  
    }  
  
};  
```  
  
## <a name="example"></a>Exemplo  
 O código a seguir mostra uma solução ao exemplo anterior.  
  
```cpp  
  
struct Account  
{  
    _Guarded_by_(cs) int balance;  
    CRITICAL_SECTION cs;  
  
    _No_competing_thread_ void Init()  
    {  
        balance = 0; // OK  
    }  
  
    _Requires_lock_held_(this->cs) void FuncNeedsLock();  
  
    _No_competing_thread_ void FuncInitCallOk()  
        // this annotation requires this function is called   
        // single-threaded, therefore we don't need to worry   
        // about the lock  
    {  
        FuncNeedsLock(); // OK, single threaded  
    }   
  
    void FuncInitCallBadFixed() // this function now properly acquires (and releases) the lock  
    {  
        EnterCriticalSection(&this->cs);FuncNeedsLock();LeaveCriticalSection(&this->cs);  
    }  
};  
  
```
